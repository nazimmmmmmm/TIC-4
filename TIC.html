Chapitre 1 : Introduction

1) Les techniques de l’information et de la communication (TIC) :
• Regroupent l'ensemble des outils pour manipuler, traiter, produire, faire circuler, transmettre, modifier et échanger de l'information
• Permettent une meilleure diffusion de l'information et de la conserver
• Regroupent l'informatique, les télécommunications, l'électronique et l'audiovisuel
2) Catégories et objectifs
• Les TIC sont présentées dans 3 grandes catégories : téléinformatique, télécommunication et électronique
• Objectifs recherchés : connectivité, modularité, implantation simple, utilisation facile, fiabilité
3) Composants de TIC
 • La partie matérielle : PC portable, téléphone portable, tablette interactive, etc
• La partie logicielle : systèmes d'exploitation, logiciels d'application et d'utilisation des différents composants matériels, réseaux d'internet ou mobiles
4) Caractéristiques
 • La rapidité : permet de réaliser des activités plus rapidement sans déplacement des agents
• La décentralisation du pouvoir : permet aux agents de recevoir les informations se référant à son domaine grâce aux forums de discussion pour arriver à la performance
• Un meilleur accès aux informations : facilite le partage de ces informations pour une plus grande performance de l'organisation
• La rationalité : avantage pour l'entreprise pour le partage des informations confidentielles ne concernant que deux niveaux hiérarchiques
• La mondialisation : participe à l'activité de la mondialisation économique
• La répartition : l'information circule entre un nombre très élevé de stations avec une qualité excellente
• L'intensité : on peut transmettre et recevoir des informations dans n’importe quel temps et de n’importe quelle région
5) Initiation aux techniques de communication
 1-Objectifs de communication :
 • Les objectifs de communication visent les réactions attendues du récepteur
• Un objectif de communication doit être défini selon trois dimensions : cognitif, affectif et comportemental
2-Système de communication :
 • Un système de communication comprend les partenaires, les vecteurs, les fonctions et le bruit
• Les partenaires de communication sont l'émetteur et le récepteur
• Les vecteurs de communication sont le message et les médias
• Les fonctions de communication sont le codage, le décodage, la réponse et le feedback
• Le bruit peut perturber la transmission des signaux
3-Types de communication :
 • La communication institutionnelle (corporate) concerne l'image de marque de l'entreprise ou de l'institution
• La communication commerciale ou de produit met en avant les caractéristiques, les performances et les avantages du produit
4-Schémas de communication :
 • La communication interpersonnelle est une communication entre un émetteur et un destinataire
• La communication de groupe est utilisée pour communiquer un message à plusieurs destinataires
• La communication de masse est utilisée pour diffuser un message à un large public
5-Média et Multimédia :
 • Les médias sont des moyens impersonnels permettant la diffusion large et collective d'informations
• Le multimédia est l'utilisation simultanée et interactive de plusieurs modes de représentation de l'information (textes, sons, images fixes ou animées)

Chapitre 3: Le modèle OSI

Les composants matériels d'un réseau informatique :
Les cartes réseau (NIC) permettent à un ordinateur de se connecter à un réseau. Elles transmettent et reçoivent des données via des protocoles de communication spécifiques.
Les switches (commutateurs réseau) connectent plusieurs appareils sur un réseau local (LAN) et permettent une communication efficace entre eux.
Les hubs (concentrateurs) connectent également plusieurs appareils sur un LAN, mais ils diffusent les données à tous les appareils, sans distinction.
Le modèle OSI :
Le modèle OSI divise les communications en sept couches distinctes, chacune ayant des fonctions spécifiques liées à la communication de données entre les appareils de réseau. Les couches vont de la couche physique (transmission des données sur le support physique) à la couche application (communication entre les applications sur les appareils de réseau).
Le modèle TCP/IP :
Le modèle TCP/IP est un modèle de référence pour les communications de réseau. Il divise les communications en quatre couches principales : la couche liaison de données, la couche réseau, la couche transport et la couche application. Il est utilisé dans la conception et la mise en œuvre des réseaux informatiques modernes.
L'adressage IP :
L'adressage IP est un système permettant d'identifier de manière unique chaque ordinateur dans un réseau. Il existe deux modes d'adressage : IPv4 et IPv6. Les adresses IPv4 sont composées de quatre octets (32 bits) et sont représentées en décimale, séparées par des points. Les adresses IPv6 sont codées sur 128 bits et sont représentées en hexadécimal.
L'adresse IPv4 :
Une adresse IPv4 est divisée en deux parties : l'adresse réseau et l'adresse hôte. Les bits de la partie réseau sont les mêmes pour tous les hôtes d'un réseau, tandis que les bits de la partie hôte déterminent le nombre de machines possibles sur ce réseau. Les adresses IPv4 peuvent être des adresses réseau, des adresses de diffusion ou des adresses d'hôtes

Chapitre 5: Système de Gestion de Version

1) Introduction:
- Définition de Git et GitHub.
- L'importance des systèmes de gestion de version.
2) Qu'est-ce que Git?.
- Logiciel de versioning créé par Linus Torvalds en 2005.
- Permet de conserver un historique des modifications et de revenir à des versions antérieures.
3) L'utilité d'un système de gestion de version:
- Gestion des modifications d'un projet.
- Collaboration efficace en équipe.
- Coexistence de différentes versions de fichiers.
4) Les deux modèles des logiciels de gestion de version: centralisé vs décentralisé:
- Modèle centralisé: code source hébergé sur un serveur central.
- Modèle décentralisé (Git): chaque utilisateur possède une copie complète du code.
5) Qu'est-ce que GitHub?
- Service en ligne pour héberger des dépôts Git.
- Le plus grand hébergeur de dépôts Git.
6) Utilisation de Git:
-ligne de commande, console et interface graphique.
- Utilisation de la ligne de commande pour accéder à toutes les commandes Git.
7) Installation de Git:
- Téléchargement de la dernière version depuis le site officiel et installation.
8) Paramétrage de Git:
- Configuration des données utilisateur (nom et adresse e-mail).
9) Démarrage d'un dépôt Git:
- Importation d'un répertoire existant ou clonage d'un dépôt existant.
10) La gestion des informations selon Git:
- Git considère les données comme des instantanés.
- Chaque instantané est stocké localement dans une base de données.
11) Les états des fichiers:
- Fichier suivi ou non suivi.
- Fichier modifié, indexé ou validé.
12) Les zones de travail:
- Répertoire de travail, zone d'index et répertoire Git.
- Extraction des fichiers, stockage des informations d'instantané, et stockage des méta-données et objets dans le répertoire Git.
13) Les branches:
-Les branches sont un concept essentiel dans Git. Elles permettent de travailler sur des versions distinctes du projet en parallèle, sans affecter la branche principale (habituellement appelée branche "master" ou "main"). Les branches facilitent la collaboration, l'expérimentation et la gestion des fonctionnalités. On peut créer de nouvelles branches, basculer entre les branches existantes et fusionner les modifications d'une branche dans une autre.
14) Les opérations de base avec Git:
-Cette section explique les commandes de base pour travailler avec Git. Elle couvre les commandes pour initialiser un dépôt, cloner un dépôt existant, effectuer des modifications, ajouter des fichiers à l'index, valider les modifications, afficher l'historique des commits et récupérer des versions précédentes du projet.
15) GitHub et les dépôts distants:
-GitHub est une plateforme d'hébergement de dépôts Git. On peut utiliser Git localement et se connecter à un dépôt distant sur GitHub pour partager son code avec d'autres développeurs. Cette section explique comment créer un dépôt sur GitHub, y pousser des modifications locales, cloner un dépôt distant et collaborer avec d'autres contributeurs.
16) Les conflits de fusion:
-Lorsque deux branches ont apporté des modifications concurrentes à un même fichier, il peut y avoir des conflits de fusion lors de la tentative de les fusionner. Cette section explique comment résoudre les conflits de fusion en identifiant les zones en conflit, en modifiant manuellement le fichier pour résoudre les différences et en finalisant la fusion.
17) Les fonctionnalités avancées de Git:
-Cette section aborde certaines fonctionnalités avancées de Git, telles que les tags (étiquettes) pour marquer des versions spécifiques du projet, les sous-modules pour intégrer des dépôts Git en tant que sous-dossiers d'un autre dépôt, les alias de commandes personnalisées pour simplifier les tâches fréquentes, et les hooks Git pour automatiser des actions lors de certaines opérations.
